# چیدمان حافظه

Sinux در نشانگاه سنتی **۱ مگابایت** بارگذاری می‌شود و حافظه‌ی فیزیکی بالای هسته را از طریق یک مدیر حافظه‌ی فیزیکی مبتنی بر بیت‌مپ (PMM) مدیریت می‌کند.

```
0x000000 - 0x0FFFFF    حافظه‌ی پایین (BIOS، رزرو شده)
0x100000               آدرس بارگذاری هسته (۱ مگابایت)
0x200000               بیت‌مپ PMM
[kernel BSS]           جداول صفحه‌بندی، پشته‌ی هسته (۳۲ کیلوبایت)
[بالای هسته]          صفحات فیزیکی آزاد (مدیریت‌شده توسط PMM)
/                      ramfs (در RAM، با ری‌بوت از بین می‌رود)
/mnt/disk              ext2 روی درایو ATA (پایدار)
/proc                  procfs (اطلاعات هسته)
```

## نکات

- **بیت‌مپ PMM** پیگیری می‌کند کدام صفحات فیزیکی آزاد یا استفاده‌شده هستند.
- پشته‌ی هسته **۳۲ کیلوبایت** است.
- `/` توسط **ramfs** پشتیبانی می‌شود و بنابراین فرار است — محتویات آن با ری‌بوت از بین می‌رود.
- `/mnt/disk` توسط **ext2** روی یک درایو ATA پشتیبانی می‌شود و ذخیره‌سازی پایدار فراهم می‌کند.
- `/proc` اطلاعات هسته را از طریق **procfs** نمایش می‌دهد.

برای نحوه‌ی فعال‌سازی صفحه‌بندی پیش از اجرای هسته، [روند بوت](./boot-flow) را ببینید.
